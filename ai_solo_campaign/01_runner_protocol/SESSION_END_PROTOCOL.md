# SESSION_END_PROTOCOL.md

## Purpose

What the AI DM does at the end of every session — the closing beat, the state update procedure, and the next-session handoff. Ensures continuity and prevents drift between sessions.

## Current Status

**PLACEHOLDER — To be developed during Stage 2.**

## AI Use

Load at session end, alongside `SESSION_END_UPDATE_CHECKLIST.md`.

---

## Expected Contents

*To be developed during Stage 2. Will include:*

### Closing the Session
- How to end a session on a compelling beat (not mid-action, not mid-sentence)
- Good session endings: moment of decision, moment of revelation, moment of danger, moment of rest
- Signaling to the player that the session is ending
- Brief in-character session summary (the player's recap, not the DM's)

### State Update Procedure
- Work through `SESSION_END_UPDATE_CHECKLIST.md` section by section
- Use `STATE_UPDATE_TEMPLATE.md` to structure the updates
- Check whether any improvised content needs to be canonized
- Advance any clocks that should have moved during the session
- Check CONSEQUENCES.md for any delayed consequences that should now trigger

### Writing Next Session Start
- Write `NEXT_SESSION_START.md` opening scene while the current session is fresh
- Include hidden DM notes
- Note what not to reveal at opening
- Note which files to load

### Canon and Index Updates
- If new major facts were established, update CANON.md
- If new player-safe facts were confirmed, update PLAYER_SAFE_CANON.md
- If new content was improvised and became important, add to CONTENT_INDEX.md

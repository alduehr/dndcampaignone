# _PLACEHOLDER.md — /05_regions/

## Purpose

This folder will contain one file per region in the campaign world.

## Current Status

**Empty — Stage 0 scaffold.** Region files will be created during Stage 3 (Starting Region Deep Build).

## Expected Contents

Each region file should follow the Region Standard defined in `00_control/WORLDBUILDING_STANDARDS.md`:

- One-sentence identity
- Player-facing overview
- DM-only truth
- Geography
- Major and minor settlements
- Wilderness zones
- Travel routes
- Factions present
- Regional conflicts
- Regional secrets
- Recurring threats
- Quest hooks
- What changes if ignored

## File Naming Convention

`[region-name].md` — e.g., `ashfield-lowlands.md`, `the-grey-reach.md`

## Stage 1 Target

The starting region identity will be established during Stage 1. Full regional development happens in Stage 3.
